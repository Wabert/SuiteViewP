# Sample File Sources — Audit testing

Generated by `tools/make_sample_file_sources.py`. An internally-consistent
insurance dataset: every claim's `PolicyNumber` exists in `POLICIES.csv`.

## Files

| File | Format | Use it to test |
|---|---|---|
| `POLICIES.csv` | CSV, header | A second File Source to JOIN against claims |
| `direct/CLAIMS.csv` | CSV, header | The base claims file |
| `reinsurance/RGACLAIMS.csv` | CSV, header | **Same layout, different folder** — add it to the CLAIMS source as a 2nd member (each is its own table); UNION them in SQL |
| `variants/CLAIMS_no_header.txt` | comma, **no header** | Choose Delimited → no header, then **Name Columns** |
| `variants/CLAIMS_pipe.txt` | `|`-delimited | Pick a non-comma delimiter |
| `variants/CLAIMS_fixed_width.txt` | Fixed width | The fixed-width layout (spec below) |

Claim columns: PolicyNumber, ClaimNumber, InsuredName, State, ClaimType, ClaimAmount, Status, DateReported, DateClosed

(`DateClosed` is blank for Open/Pending claims — handy for null tests.)

## Suggested walkthrough

1. New Query → **File Source**. Drag `direct/CLAIMS.csv` on → it sets the
   format + columns. Drag `reinsurance/RGACLAIMS.csv` on → added as a 2nd
   table (validated against the schema). Save.
2. **SQL Query →** and try a cross-file roll-up:

   ```sql
   SELECT State, ClaimType, COUNT(*) AS claims, SUM(CAST(ClaimAmount AS DOUBLE)) AS total
   FROM (SELECT * FROM "CLAIMS" UNION ALL SELECT * FROM "RGACLAIMS")
   GROUP BY State, ClaimType
   ORDER BY total DESC
   ```

3. Make a second File Source from `POLICIES.csv`, then join claims to
   policies on `PolicyNumber` (Manual SQL, or in DataForge).

## Fixed-width column spec

In the File Source editor, choose **Fixed width** and paste these
`name,start,width` lines (1-based start):

```
PolicyNumber,1,8
State,9,2
ClaimType,11,10
ClaimAmount,21,12
Status,33,8
DateReported,41,10
```
